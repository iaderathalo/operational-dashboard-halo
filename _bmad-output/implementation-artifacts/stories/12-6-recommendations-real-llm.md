# Story 12.6: Wire the maturity recommendations to a real LLM provider (replace the grounded mock)

Status: backlog

## Story

As a **user**, I want the maturity recommendations generated by a **real LLM** (GPT / Copilot /
Lenai), so that the suggestions are model-authored rather than the deterministic mock â€” while
keeping the **same grounded, validated output contract** as `12-2`.

## Context / Why

`12-2` (done) shipped a **provider-agnostic** recommendations module: a deterministic grounded
**mock** client + a **grounding validator** that rejects/repairs hallucinated values, behind
`RECOMMENDATIONS_LLM_PROVIDER` with a pluggable `RecommendationLlmClient` (`real-llm-client.ts` is
the stub). In the **2026-06-24 Bernie/Saule sync**: Saule to share a **Postman collection / API
key** for a POC; present the feature to get **formal access** to a GPT/Copilot API. Bernie also
noted Datadog has an **MCP** and that GPT is good at generating Datadog **queries / monitor
templates** (a related, separate idea worth a spike).

## Acceptance Criteria

1. A **real LLM client** implements `RecommendationLlmClient`, selected when
   `RECOMMENDATIONS_LLM_PROVIDER` is set; the **grounded mock remains the fallback** when
   unset/unavailable.
2. The model is called with the deterministic **FACTS block** + a **STRICT JSON schema**; the
   existing **validator rejects/repairs** any ungrounded value (hallucination), same bar as the
   mock.
3. API **key via Vault**; per-app **cache by `basedOnSyncAt`** + **rate-limit** (per the `12-2`
   design).
4. Output **shape identical** (`RecommendationResult`); recommendations stay **grounded only** (no
   invented metrics). Freshness-honest.
